### 3. Design Inspiration: 
As present in the side-by-side view of the high fidelity wireframes linked above, the various apps that have been considered for the design of the system are as follows:
- Deputy - Dashboard and Daily TimeTable
- Ajack - Horizontal course layout and Monthly TimeTable
- Attendance Manager - Admin Attendance View

### 4. Usability principles:
- Strive for consistency: The dark teal color scheme, top header, card shapes and homogeneous navigation bars are reused identically across every screen, thus, allowing for consistency.
- Provide appropriate feedback: Every action produces visible state change such as the green "Attendance saved" card, the conflict warning on slot addition and color-coding (green/amber/red) for manual attendance override.
- Reduce Short Term Memory (STM) load: Course codes, room names, and percentages are always shown inline users never need to remember them across screens. The timetable groups slots by day tab so the user only sees one day's load at a time unless chosen to do so otherwise.
